# ADR 0006: キャプチャ UI — ローカル優先（Inbox キュー ＋ Claude Code Channels）、クラウド先送り

- Status: Accepted
- Date: 2026-06-28
- Deciders: Project owner (ttk1010), Lead Software Engineer (Claude Code)

## Context

将来のインターフェースとして、モバイル・PC 双方から日常的に AI 概念やニュースを投入し Vault に連携したい。重要な制約は **Vault と OpenAI キーがローカルにある**こと：入力場所（モバイル/アプリ）と処理場所（キーと Vault があるマシン）が分離している。

橋渡しの方式がコスト構造を決める。常時起動のクラウドサーバ（FastAPI 等）は固定ホスティング費・クラウドへのキー配置・濫用（課金暴走）リスクを伴う。ユーザーはコスト最小化を強く志向し、Claude Pro サブスクを保有している。

## Decision

### A. 処理はローカル、固定費ゼロ

キャプチャと処理をキューで分離し、**処理はすべてローカルで行う**。常時起動サーバを持たず、OpenAI キーをクラウドに置かない。運用コストは「OpenAI のノート生成分のみ」とし、固定の月額ホスティング費を発生させない。

- Vault の `00 Inbox/` をキャプチャキューとする（Obsidian Sync / iCloud / git で同期）。
- ローカルのワーカー（ファイル監視 or 定期実行）が PC 起動時にまとめて処理する。投入は常時可能、処理は PC オン時に遅延実行。

### B. チャット投入は Claude Code Channels を採用

チャット経由の投入は **Claude Code Channels（Telegram）** を用いる。メッセージ → ローカルで起動中の Claude Code → 既存 `asb` CLI を実行 → Vault へ書き込み。自作サーバ不要で、ユーザーの Claude サブスク内で動く（ADR 0005 と同じ思想）。

### C. クラウドサービス（FastAPI 等）は先送り

「PC オフでも即時処理」を可能にするクラウドサービスは、固定費・クラウドキーのセキュリティ・濫用対策（認証・レート制限・日次上限）を要するため **当面スコープ外**とする。常時処理が必須要件になった時点で再検討する。

## Consequences

- スマホ/PC 双方から 1 分未満でキャプチャでき、固定の月額費が増えない（コストは従量の OpenAI 生成分のみ）。
- Claude Code Channels の採用で、チャット投入を自作ワーカーなしに実現できる（既存 CLI を再利用）。
- トレードオフ: 処理は **PC が起動・オンラインの時にまとめて**行われる（投入は常時可能だが処理は遅延）。即時処理が要るユースケースには応えない。
- Channels はリサーチプレビュー機能であり、Claude サブスクとローカルのターミナル常駐を前提とする。リモートからの許可プロンプト承認は不可のため、`asb` 実行コマンドは事前許可しておく。

## Alternatives considered

- **常時起動のクラウドサーバ（FastAPI）＋クラウド同期 Vault**: PC オフでも処理できる利点はあるが、固定ホスティング費・クラウドへのキー配置・濫用による課金暴走リスクを伴い、最小コスト要件に反するため先送り。
- **自作の Telegram Bot API ワーカー（long-poll）**: Claude サブスク非依存で固定費ゼロだが、トランスポートを自作する手間がある。既に Claude Code を使う前提なら Channels の方が構築コストが小さいため、第一候補は Channels とする（自作ワーカーは代替として残す）。
- **LINE Bot**: 日本では一般的だが Webhook（公開エンドポイント）必須でサーバが要る＝固定費・濫用面が増えるため、固定費ゼロ方針では不採用。
